## 分析框架
你需要始终遵循以下四个步骤，严谨地解构复杂代码库。
<guidance>

### 第一步：高层愿景与价值（“为什么”）
首先确立仓库的战略背景。在深入代码之前，回答以下基础性问题。
*   **核心目的与价值主张：**
    *   这个仓库具体解决什么问题？请清晰、简洁地阐述。
    *   对于投入时间研究这个代码库的开发者而言，可以期待学到哪些关键收获和可迁移技能？

### 第二步：架构深度剖析（“是什么”与“如何做”）
解构仓库的结构与实现。聚焦于技术设计及其如何达成目的。
*   **架构概览：**
    *   描述高层架构。
    *   核心模块或目录有哪些？为每个部分定义其单一职责。
*   **关键模块与实现：**
    *   识别出构成该仓库核心的 2 至 3 个最关键模块。
    *   这些关键模块之间是如何交互的？

### 第三步：面向受众的分析（“为谁服务”）
根据文档的最终用户来调整你的分析。识别主要受众：
*   **前端开发者：** UI 组件、框架集成、状态管理、性能优化。
*   **后端开发者：** API 设计、数据库模式、可扩展性、安全性、并发处理、部署。
*   **算法工程师/研究员：** 核心算法的正确性、效率、数学基础。
*   **学习者/学生：** 清晰的解释、分步教程、逻辑递进。

### 第四步：综合与结构化输出（“如何呈现”）
现在，将你的发现整合为一份结构清晰的最终文档目录。
*   **结构规则：**
    *   **创建逻辑层级：** 使用清晰、描述性的标题。
    *   **抽象归纳，而非镜像罗列：** 不要将文件或文件夹名称直接作为标题。应创建有意义的主题标题。
    *   **简洁且准确：** 确保每个标题都能完美概括对应章节的内容。
*   **最终输出结构：**
1. 将大纲组织为严格按如下定义的 **章节**：
   - `Get Started`：入门内容，快速上手（教程、环境搭建、使用方法）
     - 此章节下的前两个主题必须是：
       - `Overview`：对项目是什么以及为何重要的高度概括
       - `Quick Start`：运行或试用项目的分步设置指南
   - `Deep Dive`：技术详解与参考资料（概念、架构、API）
2. 在每个 `<section>` 内，你可以包含 `<topic level="...">` 和可选的 `<group>` 来聚合相关主题。
   - 每个主题必须包含其难度级别：Beginner（初级）、Intermediate（中级）或 Advanced（高级）
3. **主题总数不得超过 30 个。** 优先考虑最重要的主题。合并或省略较次要的主题，以确保总数不超限。
</guidance>
